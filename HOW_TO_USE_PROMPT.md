# 如何使用 PROMPT.md

## 📖 文档说明

`PROMPT.md` 是 **Job Hunter V2 ETL系统** 的完整任务执行指南,包含了从零开始构建整个ETL系统所需的所有信息。

## 🎯 适用场景

### 场景1: 新团队成员加入
> "我刚加入项目,如何快速了解ETL系统架构?"
→ 阅读 PROMPT.md 的「项目概述」和「项目结构」章节

### 场景2: 添加新网站ETL
> "我需要为一个新的招聘网站添加ETL脚本,应该怎么做?"
→ 遵循 PROMPT.md 的「扩展新网站」章节,按步骤执行

### 场景3: 代码审查
> "我需要审查ETL脚本,检查是否符合规范"
→ 参考 PROMPT.md 的「技术要求」和「验证标准」章节

### 场景4: 问题排查
> "ETL脚本运行出错了,如何排查?"
→ 查看 PROMPT.md 的「常见问题」和「最佳实践」章节

## 🚀 快速开始

### 1. 首次阅读 (30分钟)
- 阅读「项目概述」了解任务背景
- 阅读「数据结构规范」理解输出格式
- 阅读「ETL脚本编写规范」掌握核心方法

### 2. 实践操作 (1小时)
- 按照「执行步骤」完成第一个ETL脚本
- 使用「验证标准」检查输出质量
- 参考「示例网站」对比实现

### 3. 深入学习 (持续)
- 阅读「最佳实践」提升代码质量
- 学习「扩展新网站」添加更多网站
- 参考「常见问题」解决实际问题

## 💡 使用技巧

### 按需阅读
- **快速上手**: 只读「执行步骤」章节
- **深入理解**: 完整阅读「技术要求」和「ETL脚本编写规范」
- **问题解决**: 查阅「常见问题」和「最佳实践」

### 结合实践
- 边读边写: 打开 `etl_template.py` 对照学习
- 对比分析: 查看 `etl_didi.py` 等已完成脚本
- 测试验证: 使用 `test_etl.py` 验证代码

### 版本控制
- PROMPT.md 随项目演进持续更新
- 查看文件头部的「版本」和「更新时间」
- 定期同步最新内容
